# Nilebooks - نسخة الموبايل (Android)

تطبيق حسابات كامل لهاتف أندرويد، مبني بـ Python (Kivy + KivyMD)، مستقل تماماً عن نسخة سطح المكتب
(بيانات منفصلة على كل جهاز). يعمل بدون إنترنت وبقاعدة بيانات محلية SQLite داخل الهاتف.

## المميزات
نفس مزايا نسخة سطح المكتب: لوحة تحكم، مخزون مع تنبيه النقص، فواتير بيع وشراء تُحدّث المخزون
تلقائياً، عملاء وموردون بأرصدة، مصاريف وإيرادات، تقارير بفلترة تاريخ، وعملات متعددة.

---

## ⚠️ خطوة مهمة قبل البناء: إضافة خط عربي

Kivy لا يأتي بخط يدعم العربية بشكل كامل، لذلك **يجب** إضافة ملف خط قبل بناء التطبيق:

1. حمّل خط **Noto Naskh Arabic** المجاني من Google Fonts:
   https://fonts.google.com/noto/specimen/Noto+Naskh+Arabic
2. ضع ملف `NotoNaskhArabic-Regular.ttf` داخل مجلد:
   ```
   assets/fonts/NotoNaskhArabic-Regular.ttf
   ```
   (المسار مطابق تماماً لما هو مكتوب في `arabic_support.py`)
3. ارفع هذا الملف مع باقي الكود إلى GitHub (لا تتجاهله في `.gitignore`).

بدون هذا الملف، التطبيق سيعمل لكن النصوص العربية ستظهر بحروف غير متصلة بشكل صحيح.

---

## بناء ملف APK تلقائياً عبر GitHub Actions (بدون أي إعداد على جهازك)

1. أنشئ مستودع (Repository) جديد فارغ على GitHub.
2. ارفع محتويات هذا المجلد بالكامل إليه (تأكد من رفع مجلد `.github/workflows` ومجلد `assets/fonts` بالخط):
   ```
   git init
   git add .
   git commit -m "Nilebooks mobile - initial commit"
   git branch -M main
   git remote add origin <رابط-المستودع-الخاص-بك>
   git push -u origin main
   ```
3. بمجرد الرفع، ستبدأ عملية البناء تلقائياً. لمتابعتها:
   - افتح المستودع على GitHub → تبويب **Actions**
   - ستجد عملية باسم "Build Android APK" قيد التنفيذ (تستغرق 15-25 دقيقة أول مرة)
4. عند اكتمال العملية بنجاح (علامة ✅ خضراء):
   - افتح تفاصيل العملية → قسم **Artifacts** أسفل الصفحة
   - حمّل ملف `Nilebooks-APK.zip`، وبداخله ملف `.apk` جاهز للتثبيت
5. انقل ملف الـ APK إلى الهاتف وثبّته (فعّل "تثبيت من مصادر غير معروفة" في إعدادات الأندرويد إن طُلب ذلك).

### إعادة البناء لاحقاً
أي رفع (`git push`) جديد لفرع `main` يشغّل البناء تلقائياً من جديد. يمكنك أيضاً تشغيله يدوياً من
تبويب Actions عبر زر **Run workflow** (لأن الإعداد يتضمن `workflow_dispatch`).

## هيكل المشروع
```
nilebooks_mobile/
├── main.py                    # نقطة البدء + القائمة الجانبية وربط الشاشات
├── database.py                 # قاعدة البيانات المحلية (SQLite)
├── arabic_support.py            # تشكيل النصوص العربية لعرضها بشكل صحيح في Kivy
├── ui_helpers.py                # عناصر واجهة قابلة لإعادة الاستخدام (قوائم، نوافذ نماذج)
├── screen_dashboard.py           # لوحة التحكم
├── screen_products.py            # المنتجات والمخزون
├── screen_contacts.py             # العملاء والموردون
├── screen_invoices.py             # فواتير المبيعات والمشتريات
├── screen_expenses.py              # المصاريف والإيرادات
├── screen_reports.py               # التقارير
├── screen_settings.py              # الإعدادات والعملات
├── buildozer.spec                   # إعدادات بناء تطبيق أندرويد
├── requirements.txt
├── assets/fonts/                    # ضع هنا خط Noto Naskh Arabic (انظر أعلاه)
└── .github/workflows/build-apk.yml  # يبني APK تلقائياً على GitHub
```

## التجربة على الكمبيوتر قبل البناء للهاتف (اختياري)
```
pip install -r requirements.txt
python main.py
```
هذا يفتح نافذة على الكمبيوتر تحاكي شكل التطبيق على الهاتف، مفيد لتجربة سريعة قبل بناء APK كامل.

## ملاحظات مهمة
- **البيانات منفصلة عن نسخة سطح المكتب** — كل جهاز له قاعدة بياناته الخاصة. لو احتجت مزامنة
  البيانات بين الجهازين مستقبلاً، هذا يحتاج خادم/قاعدة بيانات سحابية إضافية (خطوة تالية منفصلة).
- عند أول تشغيل، التطبيق يُنشئ عملة افتراضية (ريال سعودي) يمكنك تعديلها من شاشة الإعدادات.
- نسخة الموبايل لا تتضمن الرسوم البيانية (تُعرض كأرقام ملخصة فقط) لتبسيط الاعتماديات؛ يمكن
  إضافتها لاحقاً إذا رغبت.

## أفكار للتطوير قبل البيع
- إضافة أيقونة مخصصة (`assets/icon.png` مقاس 512×512) وتفعيل السطر الخاص بها في `buildozer.spec`.
- نظام تفعيل/ترخيص لحماية النسخ المباعة.
- طباعة/تصدير الفواتير كـ PDF مباشرة من الهاتف.
- مزامنة سحابية بين نسخة الكمبيوتر والموبايل (تحتاج خادم backend).
